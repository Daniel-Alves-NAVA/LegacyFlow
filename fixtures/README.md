# Fixtures de Avaliação

Esta pasta reúne exemplos mínimos para validar o comportamento do harness.

## Casos recomendados

- fluxo REST simples;
- listener Kafka;
- job batch;
- fluxo com dependência externa e tópico configurado.

## Objetivo

Garantir que mudanças no kit não quebrem a geração de documentação nem a revisão.
