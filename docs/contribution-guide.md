# Guia de Contribuição

## Como contribuir

1. Crie ou ajuste uma skill, prompt, template ou instruction.
2. Mantenha o padrão de Markdown claro e objetivo.
3. Evite adicionar conteúdo que invente comportamento.
4. Atualize a documentação relevante quando a estrutura mudar.
5. Teste a instalação com Gradle antes de enviar mudanças.
